What is a Design Pattern?

A Design Pattern is a reusable solution to a common software design problem.

Instead of solving the same problem every time, developers use design patterns because they are already tested and proven solutions.

Why do we use Design Patterns?
Makes code reusable
Makes code easier to maintain
Reduces code duplication
Makes applications flexible and scalable
Improves communication among developers

1. Singleton Pattern
Definition

Ensures that only one object of a class is created throughout the application.

Why use it?

When only one shared object is needed.

Real-life Example

There is only one Principal in a school.

Uses
Database Connection
Logger
Configuration Manager
Key Point

✔ Only one instance exists.

2. Factory Pattern
Definition

Creates objects without exposing the object creation logic.

The Factory decides which object should be created.

Why use it?

When different objects are created based on input.

Real-life Example

A Pizza Shop.

You ask for Pizza.

The shop decides whether to make Veg Pizza or Cheese Pizza.

Uses
Payment Systems
Notification Systems
UI Components
Key Point

✔ Factory creates objects.

3. Builder Pattern
Definition

Builds complex objects step by step.

Why use it?

When an object has many optional fields.

Real-life Example

Ordering a Burger.

Choose:

Bread
Cheese
Sauce
Vegetables

The same process creates different burgers.

Uses
Creating User objects
Building HTTP Requests
Complex Configurations
Key Point

✔ Builds objects gradually.


4. Adapter Pattern
Definition

Allows two incompatible classes to work together.

Why use it?

When existing classes have different interfaces.

Real-life Example

A mobile charger adapter.

Phone charger and wall socket are different, but the adapter connects them.

Uses
Third-party APIs
Legacy systems
Key Point

✔ Converts one interface into another.

5. Decorator Pattern
Definition

Adds new functionality to an object without changing its original code.

Why use it?

To extend object behavior dynamically.

Real-life Example

Coffee

Coffee → Coffee + Milk → Coffee + Milk + Sugar

The coffee stays the same, but features are added.

Uses
Text Formatting
Java I/O Streams
Notifications
Key Point

✔ Adds extra features.

6. Observer Pattern
Definition

When one object changes, all dependent objects are automatically notified.

Why use it?

When multiple objects need updates.

Real-life Example

You subscribe to a YouTube channel.

Whenever a new video is uploaded,

All subscribers receive notifications.

Uses
Notifications
Chat Applications
Stock Market Apps
Key Point

✔ One changes → Many get notified.

7. Strategy Pattern
Definition

Provides multiple algorithms, and one can be selected at runtime.

Why use it?

When different methods solve the same problem.

Real-life Example

Google Maps

Choose

Car
Bike
Walking

Different routes for different strategies.

Uses
Payment Methods
Sorting Algorithms
Travel Routes
Key Point

✔ Choose one algorithm from many.

8. Facade Pattern
Definition

Provides one simple interface to a complex system.

Why use it?

To hide system complexity.

Real-life Example

Remote Control

Press one button.

TV performs many internal operations.

Uses
Home Theater System
Banking APIs
Spring Boot Services
Key Point

✔ One simple interface for many operations.
