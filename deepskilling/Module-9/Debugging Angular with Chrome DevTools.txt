1. Setting Up the Environment (ng serve)

Angular applications are run using the Angular CLI.

Step 1: Install Angular CLI
npm install -g @angular/cli
Step 2: Navigate to the Project
cd MyAngularApp
Step 3: Start the Development Server
ng serve

or

ng serve --open

--open automatically opens the browser.

Output
✔ Browser application bundle generation complete.

Local: http://localhost:4200/

Open:

http://localhost:4200

Whenever you save a file, Angular automatically recompiles and refreshes the browser (Hot Reload).

2. Opening Chrome DevTools (F12)

Chrome DevTools is a built-in debugging tool in Google Chrome.

Ways to Open DevTools
F12
Ctrl + Shift + I
Right-click → Inspect

DevTools contains several useful tabs:

Tab	Purpose
Elements	Inspect HTML and CSS
Console	View logs and errors
Sources	Debug JavaScript and TypeScript
Network	Monitor HTTP/API requests
Application	View Local Storage, Session Storage, Cookies
Performance	Analyze application performance
3. Inspecting Elements (DOM Tree)

The Elements tab displays the DOM (Document Object Model) generated by Angular.

Example HTML:

<h1>Hello Angular</h1>

<button>Save</button>

DOM Tree

<html>
   └── body
        ├── h1
        └── button

You can:

Inspect HTML elements
Modify HTML temporarily
Edit CSS
View applied styles
Check element dimensions
Inspect Angular-generated DOM
Example
<div class="card">
    Student Details
</div>

DevTools displays:

div.card

You can change CSS values like:

background-color: yellow;

The changes appear immediately but disappear after refreshing the page.

5. Debugging JavaScript – Sources Panel

The Sources tab allows you to debug code step by step.

You can:

Set breakpoints
Pause execution
Inspect variables
View the call stack
Step through code
Breakpoints

A breakpoint pauses program execution at a specific line.

Example

let total = price * quantity;

console.log(total);

Click beside the line number to set a breakpoint.

When execution reaches that line, it pauses automatically.

Debugging Controls
Button	Purpose
▶ Resume	Continue execution
⏭ Step Over	Execute the current line without entering function calls
⏬ Step Into	Enter a function call
⏫ Step Out	Exit the current function
⏸ Pause	Pause execution
6. Call Stack

The Call Stack shows the sequence of function calls that led to the current line of execution.

Example

function add()
{
    calculate();
}

function calculate()
{
    display();
}

function display()
{
    console.log("Done");
}

Call Stack

display()

calculate()

add()

This helps identify where a function was called from.

8. Debugging Angular TypeScript Code (Source Maps)

Browsers execute JavaScript, but Angular developers write TypeScript.

Angular uses Source Maps to map compiled JavaScript back to the original TypeScript source, allowing you to debug the TypeScript files directly.

Without Source Maps

TypeScript

↓

JavaScript

↓

Browser

With Source Maps

TypeScript

↓

JavaScript

↓

Browser

↓

Chrome Debugger opens original TypeScript
Example

Component

export class AppComponent {

    title = "Angular";

    show()
    {
        console.log(this.title);
    }

}

Set a breakpoint on:

console.log(this.title);

Run the application.

Chrome pauses on the TypeScript line instead of the generated JavaScript.

10. Debugging an API Call

Example

this.http.get("https://localhost:5001/api/student")
.subscribe(data =>
{
    console.log(data);
});

Steps:

Set a breakpoint on the subscribe() line.
Refresh the application.
Chrome pauses when the API response arrives.
Inspect the data variable.
Continue execution.
