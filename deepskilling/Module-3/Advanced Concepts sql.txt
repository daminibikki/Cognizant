SQL Server is a Relational Database Management System (RDBMS) developed by Microsoft. It is used to store, manage, retrieve, and secure data. It supports SQL (Structured Query Language) for database operations.

OVER() Function

The OVER() clause performs calculations over a set of rows without grouping them.

Syntax

SELECT column,
       function() OVER()
FROM table_name;

Example

SELECT Name,
Salary,
AVG(Salary) OVER() AS AverageSalary
FROM Employee;
PARTITION BY

Divides rows into groups for window functions.

Syntax

SELECT column,
function() OVER(PARTITION BY column)
FROM table_name;

Example

SELECT Name,
Department,
Salary,
AVG(Salary) OVER(PARTITION BY Department) AS AvgSalary
FROM Employee;
ROW_NUMBER()

Assigns unique numbers to rows.

Syntax

ROW_NUMBER() OVER(ORDER BY column)

Example

SELECT Name,
Salary,
ROW_NUMBER() OVER(ORDER BY Salary DESC) AS RowNum
FROM Employee;

Output

Salary   RowNum
90000      1
80000      2
70000      3
RANK()

Assigns ranks.
Duplicate values get the same rank.
Next rank is skipped.

Example

SELECT Name,
Salary,
RANK() OVER(ORDER BY Salary DESC) AS Rank
FROM Employee;

Output

Salary   Rank
90000      1
80000      2
80000      2
70000      4
DENSE_RANK()

Duplicate values get the same rank.
No ranks are skipped.

Example

SELECT Name,
Salary,
DENSE_RANK() OVER(ORDER BY Salary DESC) AS Rank
FROM Employee;

Output

Salary   Rank
90000      1
80000      2
80000      2
70000      3
Difference
Function	Duplicate Rank	Skip Rank
ROW_NUMBER()	No	No
RANK()	Yes	Yes
DENSE_RANK()	Yes	No
GROUPING SETS

Produces multiple GROUP BY results in one query.

Example

SELECT Department,
Job,
SUM(Salary)
FROM Employee
GROUP BY GROUPING SETS
(
(Department),
(Job),
()
);
CUBE

Generates all possible combinations of GROUP BY columns.

Example

SELECT Department,
Job,
SUM(Salary)
FROM Employee
GROUP BY CUBE(Department,Job);

Produces

Department totals
Job totals
Department + Job totals
Grand total
ROLLUP

Creates hierarchical totals.

Example

SELECT Department,
Job,
SUM(Salary)
FROM Employee
GROUP BY ROLLUP(Department,Job);

Produces

Department + Job
Department Total
Grand Total
Difference Between CUBE and ROLLUP
ROLLUP	CUBE
Hierarchical totals	All combinations
Less output	More output
Faster	Slightly slower
WITH Statement

Used to create Common Table Expressions (CTE).

Syntax

WITH CTE_Name AS
(
SELECT ...
)
SELECT * FROM CTE_Name;
Common Table Expression (CTE)

Temporary result set.

Example

WITH EmployeeCTE AS
(
SELECT *
FROM Employee
WHERE Salary > 50000
)
SELECT *
FROM EmployeeCTE;

Advantages

Easy to read
Reusable
Simplifies complex queries
Recursive CTE

Used for hierarchical data.

Syntax

WITH CTE AS
(
Anchor Query

UNION ALL

Recursive Query
)
SELECT *
FROM CTE;

Example

WITH Numbers AS
(
SELECT 1 AS Num

UNION ALL

SELECT Num+1
FROM Numbers
WHERE Num<5
)

SELECT *
FROM Numbers;

Output

1
2
3
4
5
MERGE Statement

Used to INSERT, UPDATE and DELETE in one statement.

Syntax

MERGE TargetTable AS T

USING SourceTable AS S

ON T.ID=S.ID

WHEN MATCHED THEN
UPDATE SET T.Name=S.Name

WHEN NOT MATCHED THEN
INSERT(ID,Name)
VALUES(S.ID,S.Name);
PIVOT

Converts rows into columns.

Example Data

Department Salary

HR        1000
HR        2000
IT        3000
IT        4000

Query

SELECT *
FROM
(
SELECT Department,Salary
FROM Employee
) AS Source

PIVOT
(
SUM(Salary)
FOR Department IN ([HR],[IT])
) AS PivotTable;

Output

HR     IT

3000   7000
UNPIVOT

Converts columns into rows.

Example Data

HR    IT

3000 7000

Query

SELECT Department,Salary

FROM DepartmentSalary

UNPIVOT
(
Salary FOR Department IN (HR,IT)
) AS U;

Output

Department Salary

HR         3000
IT         7000
