What are Exceptions?
An Exception is an error or unexpected event that occurs during the execution of a SQL statement or batch. SQL Server provides exception handling mechanisms to detect errors and execute alternative code instead of terminating the program.

Examples
Divide by zero

Primary key violation

Foreign key constraint violation

Invalid column name

Data type conversion error

Table does not exist

Why Exception Handling is Needed
Prevents program termination

Handles runtime errors gracefully

Displays meaningful error messages

Maintains data integrity

Allows transactions to be rolled back

Makes debugging easier

Types of SQL Server Exceptions
SQL Server exceptions are mainly of two types:

1. System Defined Exceptions
These are automatically generated by SQL Server whenever an error occurs.

Examples
Divide by zero

Arithmetic overflow

Invalid object name

Primary key violation

Foreign key violation

Null value insertion

Conversion failed

Example

SELECT 100/0;
Output

Msg 8134, Level 16
Divide by zero error encountered.
2. User Defined Exceptions
These are custom errors generated by the programmer using THROW or RAISERROR.

Example using THROW

THROW 50001, 'Salary cannot be negative.', 1;
Output

Msg 50001
Salary cannot be negative.
Example using RAISERROR

RAISERROR('Invalid Employee ID.',16,1);
Retrieving Detailed Information on the Error
Inside a CATCH block, SQL Server provides built-in functions to retrieve error details.

Function	Description
ERROR_NUMBER()	Returns error number
ERROR_MESSAGE()	Returns complete error message
ERROR_LINE()	Returns line number
ERROR_PROCEDURE()	Returns procedure name
ERROR_SEVERITY()	Returns severity level
ERROR_STATE()	Returns state number
Example

BEGIN TRY
    SELECT 10/0;
END TRY

BEGIN CATCH
    SELECT
    ERROR_NUMBER() AS ErrorNumber,
    ERROR_MESSAGE() AS ErrorMessage,
    ERROR_LINE() AS ErrorLine,
    ERROR_SEVERITY() AS Severity,
    ERROR_STATE() AS State;
END CATCH;
Output (Example)

ErrorNumber : 8134
ErrorMessage: Divide by zero error encountered.
ErrorLine   : 2
Severity    : 16
State       : 1
TRY...CATCH
TRY...CATCH is used to handle runtime errors.

Syntax
BEGIN TRY

-- SQL Statements

END TRY

BEGIN CATCH

-- Error Handling Statements

END CATCH;
Example

BEGIN TRY

SELECT 100/0;

END TRY

BEGIN CATCH

PRINT 'An error occurred.';

END CATCH;
Output

An error occurred.
TRY...CATCH with Error Details
BEGIN TRY

SELECT 10/0;

END TRY

BEGIN CATCH

PRINT ERROR_MESSAGE();

END CATCH;
Output

Divide by zero error encountered.
TRY...CATCH with Transaction
BEGIN TRY

BEGIN TRANSACTION;

UPDATE Employee
SET Salary = Salary + 1000;

COMMIT TRANSACTION;

END TRY

BEGIN CATCH

ROLLBACK TRANSACTION;

PRINT ERROR_MESSAGE();

END CATCH;
Using Nested TRY...CATCH Constructs
A TRY...CATCH block can be placed inside another TRY or CATCH block.

Syntax

BEGIN TRY

BEGIN TRY

-- Statements

END TRY

BEGIN CATCH

-- Inner Catch

END CATCH

END TRY

BEGIN CATCH

-- Outer Catch

END CATCH;
Example

BEGIN TRY

BEGIN TRY

SELECT 10/0;

END TRY

BEGIN CATCH

PRINT 'Inner Catch';

THROW;

END CATCH

END TRY

BEGIN CATCH

PRINT 'Outer Catch';

END CATCH;
Output

Inner Catch
Outer Catch
THROW Statement
THROW is used to generate a custom exception or rethrow the current exception.

Syntax
THROW Error_Number,
'Message',
State;
Rules

Error number must be greater than or equal to 50000

State value ranges from 0 to 255

Introduced in SQL Server 2012

Preserves original error information when rethrowing

Example

THROW 50001,
'Employee salary is invalid.',
1;
Output

Msg 50001
Employee salary is invalid.
THROW inside TRY...CATCH
BEGIN TRY

SELECT 10/0;

END TRY

BEGIN CATCH

PRINT 'Exception Found';

THROW;

END CATCH;
Output

Exception Found

Divide by zero error encountered.
RAISERROR Statement
RAISERROR is used to generate custom error messages.

Syntax
RAISERROR
(
'Message',
Severity,
State
);
Example

RAISERROR
(
'Invalid Department',
16,
1
);
Output

Invalid Department
Severity Levels
Severity	Meaning
0–10	Informational message
11–16	User errors
17–19	Software or resource errors
20–25	Fatal system errors
Most user-defined errors use Severity 16.

THROW vs RAISERROR
THROW	RAISERROR
Introduced in SQL Server 2012	Available in earlier versions
Error number must be ≥ 50000	Can use custom message directly
Preserves original error details	Does not preserve original error details automatically
Simpler syntax	More options like formatting and severity
Recommended for new development	Maintained for backward compatibility
Complete Example
BEGIN TRY

DECLARE @a INT = 10;
DECLARE @b INT = 0;

SELECT @a/@b;

END TRY

BEGIN CATCH

PRINT 'Error Occurred';

SELECT
ERROR_NUMBER() AS ErrorNumber,
ERROR_MESSAGE() AS ErrorMessage,
ERROR_LINE() AS ErrorLine;

END CATCH;
Output

Error Occurred

ErrorNumber : 8134
ErrorMessage: Divide by zero error encountered.
ErrorLine   : 6
