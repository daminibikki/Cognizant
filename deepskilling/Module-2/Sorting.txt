What is Sorting?

Sorting means arranging data in a specific order.

There are two types:

Ascending Order (Small → Large)
Descending Order (Large → Small)

Example

Original:
8 3 5 1 9

Ascending:
1 3 5 8 9

Descending:
9 8 5 3 1
Why Do We Need Sorting?

Sorting makes many operations faster.

Examples:

Binary Search requires a sorted array.
Finding the smallest/largest elements becomes easier.
Data looks organized.
Used in databases, e-commerce, and search engines.
Types of Sorting Algorithms
1. Bubble Sort
Best Case: O(n)
Average Case: O(n²)
Worst Case: O(n²)
Stable: Yes ✅
In-place: Yes ✅

Used for: Small datasets and learning basic sorting concepts.

2. Selection Sort
Best Case: O(n²)
Average Case: O(n²)
Worst Case: O(n²)
Stable: No ❌
In-place: Yes ✅

Used for: Simple implementation with minimal swaps.

3. Insertion Sort
Best Case: O(n)
Average Case: O(n²)
Worst Case: O(n²)
Stable: Yes ✅
In-place: Yes ✅

Used for: Small or nearly sorted arrays.

4. Merge Sort
Best Case: O(n log n)
Average Case: O(n log n)
Worst Case: O(n log n)
Stable: Yes ✅
In-place: No ❌ (requires extra memory)

Used for: Large datasets where guaranteed performance is important.

5. Quick Sort
Best Case: O(n log n)
Average Case: O(n log n)
Worst Case: O(n²)
Stable: No ❌
In-place: Yes ✅

Used for: Fast average-case sorting; widely used in practice.

6. Heap Sort
Best Case: O(n log n)
Average Case: O(n log n)
Worst Case: O(n log n)
Stable: No ❌
In-place: Yes ✅

Used for: When guaranteed O(n log n) time and low memory usage are needed.

1. Bubble Sort
Idea

Compare adjacent elements.

If they are in the wrong order, swap them.

The largest element "bubbles" to the end after each pass.

Example

5 3 8 1

Pass 1

5 3 → Swap
3 5 8 1

5 8 → No Swap

8 1 → Swap
3 5 1 8

Largest (8) reached the end.
Java Code
for(int i=0;i<n-1;i++){
    for(int j=0;j<n-i-1;j++){
        if(arr[j]>arr[j+1]){
            int temp=arr[j];
            arr[j]=arr[j+1];
            arr[j+1]=temp;
        }
    }
}
Time Complexity

Best → O(n) (with optimization)

Average → O(n²)

Worst → O(n²)

Memory

O(1)

2. Selection Sort
Idea

Find the smallest element.

Place it at the beginning.

Repeat.

Example

7 4 9 2

Find smallest = 2

Swap with first

2 4 9 7
Java Code
for(int i=0;i<n-1;i++){

    int min=i;

    for(int j=i+1;j<n;j++){

        if(arr[j]<arr[min])
            min=j;
    }

    int temp=arr[min];
    arr[min]=arr[i];
    arr[i]=temp;
}
Time Complexity

Best

O(n²)

Average

O(n²)

Worst

O(n²)

Memory

O(1)

3. Insertion Sort
Idea

Just like arranging playing cards.

Pick one element.

Insert it into the correct position.

Example

5 2 4

Take 2

Insert before 5

2 5 4

Take 4

Insert between 2 and 5

2 4 5
Java Code
for(int i=1;i<n;i++){

    int key=arr[i];

    int j=i-1;

    while(j>=0 && arr[j]>key){

        arr[j+1]=arr[j];

        j--;
    }

    arr[j+1]=key;
}
Time Complexity

Best

O(n)

Average

O(n²)

Worst

O(n²)

Memory

O(1)

4. Merge Sort
Idea

Divide and Conquer.

Step 1

Divide array into halves.

Step 2

Sort each half.

Step 3

Merge them.

Example

8 3 6 1

↓

8 3

6 1

↓

8

3

6

1

↓

3 8

1 6

↓

1 3 6 8
Time Complexity

Best

O(n log n)

Average

O(n log n)

Worst

O(n log n)

Memory

O(n)

5. Quick Sort
Idea

Choose one element as Pivot.

Put smaller elements left.

Larger elements right.

Repeat.

Example

7 3 9 2 5

Pivot = 5

↓

3 2

5

7 9
Time Complexity

Best

O(n log n)

Average

O(n log n)

Worst

O(n²)

Memory

O(log n)
