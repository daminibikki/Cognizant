1. Authentication and Authorization in Microservices
Although these terms are often used together, they have different purposes.

Authentication
Authentication verifies who the user is.

Example:

Login using username and password.

Validate a JWT token.

Authorization
Authorization determines what the authenticated user is allowed to do.

Example:

Admin can delete users.

Customer can view only their own orders.

Flow
User
   │
Login
   │
Authentication
   │
JWT Token
   │
Authorization
   │
Access API
2. JWT (JSON Web Token)
JWT is a secure token used to authenticate users in Web APIs.

After successful login, the server generates a token and sends it to the client.

The client sends the token with every request.

JWT Structure
Header
.
Payload
.
Signature
Example:

Configure JWT in ASP.NET Core
Register Authentication
builder.Services.AddAuthentication(JwtBearerDefaults.AuthenticationScheme)
    .AddJwtBearer(options =>
    {
        // JWT configuration
    });
Enable Middleware
app.UseAuthentication();

app.UseAuthorization();
Order is important:

UseAuthentication()

UseAuthorization()

Protect Endpoints
[Authorize]
[HttpGet]
public IActionResult GetProducts()
{
    return Ok();
}
Only authenticated users can access this endpoint.

Allow Anonymous
[AllowAnonymous]
[HttpPost]
public IActionResult Login()
{
    return Ok();
}
3. Securing ASP.NET Core Web API Endpoints
Security methods include:

JWT Authentication

OAuth

HTTPS

API Keys

Role-based Authorization

Claims-based Authorization

Role-Based Authorization
[Authorize(Roles="Admin")]
public IActionResult DeleteProduct()
{
    return Ok();
}
Only users with the Admin role can access this endpoint.

Claims-Based Authorization
[Authorize(Policy="EmployeeOnly")]
Allows access based on user claims defined in the authentication token.
