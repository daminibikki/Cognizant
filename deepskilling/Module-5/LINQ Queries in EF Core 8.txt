Entity Framework Core – Querying Data with LINQ
LINQ (Language Integrated Query) is used in EF Core to query data from the database using C# instead of writing SQL.

1. Writing Queries Using Where, Select, and OrderBy
A) Where()
What is Where()?
Where() filters records based on a condition.

Example
var students = await context.Students
    .Where(s => s.Age > 20)
    .ToListAsync();
Equivalent SQL:

SELECT * FROM Students
WHERE Age > 20;
B) Select()
What is Select()?
Select() chooses specific columns instead of retrieving the entire entity.

Example
var names = await context.Students
    .Select(s => s.Name)
    .ToListAsync();
Equivalent SQL:

SELECT Name
FROM Students;
Output:

Alice
Bob
Charlie
C) OrderBy()
What is OrderBy()?
Sorts records in ascending order.

var students = await context.Students
    .OrderBy(s => s.Name)
    .ToListAsync();
Equivalent SQL:

SELECT *
FROM Students
ORDER BY Name ASC;
Descending Order
var students = await context.Students
    .OrderByDescending(s => s.Age)
    .ToListAsync();
Equivalent SQL:

SELECT *
FROM Students
ORDER BY Age DESC;
2. Projection into DTOs (Data Transfer Objects)
What is a DTO?
A DTO (Data Transfer Object) is a class that contains only the data needed by the client.

Instead of returning the complete entity, you return only selected fields.

Entity
public class Student
{
    public int Id { get; set; }

    public string Name { get; set; }

    public int Age { get; set; }

    public string Address { get; set; }
}
DTO
public class StudentDto
{
    public string Name { get; set; }

    public int Age { get; set; }
}
Projection
var students = await context.Students
    .Select(s => new StudentDto
    {
        Name = s.Name,
        Age = s.Age
    })
    .ToListAsync();
Only Name and Age are retrieved.

Advantages of DTOs
Transfers only required data.

Faster queries.

Better security (hides sensitive fields).

Reduces network traffic.

3. Filtering and Aggregating Data
Filtering
Example:

var students = await context.Students
    .Where(s => s.Age >= 18)
    .ToListAsync();
Returns only students whose age is 18 or above.

Aggregation Functions
Count
int total = await context.Students.CountAsync();
Equivalent SQL:

SELECT COUNT(*)
FROM Students;
Sum
int totalAge = await context.Students.SumAsync(s => s.Age);
Equivalent SQL:

SELECT SUM(Age)
FROM Students;
Average
double avgAge = await context.Students.AverageAsync(s => s.Age);
Equivalent SQL:

SELECT AVG(Age)
FROM Students;
Maximum
int maxAge = await context.Students.MaxAsync(s => s.Age);
Equivalent SQL:

SELECT MAX(Age)
FROM Students;
Minimum
int minAge = await context.Students.MinAsync(s => s.Age);
Equivalent SQL:

SELECT MIN(Age)
FROM Students;
4. Asynchronous Queries with ToListAsync()
What is ToListAsync()?
ToListAsync() executes the query asynchronously and returns the results as a list.

var students = await context.Students.ToListAsync();
Why use ToListAsync()?
Keeps the application responsive.

Does not block the executing thread while waiting for the database.

Recommended for ASP.NET Core web applications.

Synchronous vs Asynchronous
Synchronous
var students = context.Students.ToList();
The thread waits until the query finishes.

Asynchronous
var students = await context.Students.ToListAsync();
The thread can perform other work while waiting for the database response.

Common LINQ Methods
Method	Purpose
Where()	Filters records
Select()	Chooses specific columns
OrderBy()	Sorts ascending
OrderByDescending()	Sorts descending
ToListAsync()	Retrieves all results asynchronously
FirstOrDefaultAsync()	Returns the first matching record or null
CountAsync()	Counts records
SumAsync()	Calculates the sum
AverageAsync()	Calculates the average
MaxAsync()	Returns the maximum value
MinAsync()	Returns the minimum value
