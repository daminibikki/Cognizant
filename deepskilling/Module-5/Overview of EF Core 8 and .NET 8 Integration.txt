1. What is ORM (Object-Relational Mapping)?
ORM (Object-Relational Mapping) is a technique that maps database tables to classes and table rows to objects. It allows developers to work with databases using programming language objects instead of writing SQL for every operation.

Without ORM
You write SQL queries manually.

SELECT * FROM Students;
With ORM (Entity Framework Core)
var students = context.Students.ToList();
The ORM automatically converts this into SQL and executes it.

How ORM Works
Database Table:

Id	Name	Age
1	Alice	20
2	Bob	21
↓

C# Class

public class Student
{
    public int Id { get; set; }
    public string Name { get; set; }
    public int Age { get; set; }
}
Each row becomes an object.

Advantages of ORM
Reduces SQL code.

Faster development.

Easier maintenance.

Database-independent code.

Prevents many SQL injection issues by using parameterized queries.

Disadvantages
Can be slower than optimized SQL for complex queries.

Learning curve.

Less control over generated SQL.

2. EF Core vs EF Framework: Key Differences
Feature	Entity Framework (EF6)	Entity Framework Core (EF Core)
Platform	Windows only (.NET Framework)	Cross-platform (.NET Core/.NET 5+)
Performance	Slower	Faster
Open Source	Partially	Fully Open Source
Cross Platform	No	Yes
Cloud Support	Limited	Excellent
Dependency Injection	Limited	Built-in
LINQ Support	Yes	Yes (Improved)
Migrations	Supported	Improved and faster
Modern Features	Limited	Supports latest .NET features
EF Framework
Older ORM.

Used mainly with .NET Framework.

Suitable for legacy applications.

EF Core
Modern ORM from Microsoft.

Designed for .NET Core and newer .NET versions.

Cross-platform (Windows, Linux, macOS).

Better performance.

Recommended for new projects.

3. New Features in EF Core 8
Entity Framework Core 8 introduces several improvements.

1. Better Performance
Faster query execution.

Improved memory usage.

2. Improved SQL Generation
Generates more efficient SQL queries, reducing database load.

3. Primitive Collections
Supports collections of primitive types more naturally.

Example:

public class Student
{
    public int Id { get; set; }

    public List<string> Skills { get; set; }
}
4. JSON Column Support Improvements
Works more efficiently with JSON data stored in databases.

Example:

{
  "Name": "Alice",
  "Age": 20
}
5. Better LINQ Translation
More LINQ expressions are translated directly into SQL, reducing client-side processing.

Example:

var students = context.Students
                      .Where(s => s.Age > 18)
                      .ToList();
6. Faster Bulk Operations
Insert, update, and delete operations are optimized for better performance.

7. Better Database Compatibility
Enhanced support for databases such as:

SQL Server

SQLite

PostgreSQL

MySQL
