1. Adding Migrations
What is a Migration?
When you change an entity (for example, add a new property), EF Core creates a migration file that contains the SQL needed to update the database.

Example
Original Entity

public class Student
{
    public int Id { get; set; }
    public string Name { get; set; }
}
Suppose you add a new property:

public class Student
{
    public int Id { get; set; }
    public string Name { get; set; }
    public int Age { get; set; }
}
Create a migration:

dotnet ef migrations add AddAgeColumn
EF Core generates a migration file describing how to add the Age column.

2. Updating the Database
After creating the migration, apply it to the database.

dotnet ef database update
The database schema is updated automatically.

3. Removing Migrations
If you created a migration by mistake and have not applied it, remove it using:

dotnet ef migrations remove
This removes only the latest migration.

Note: If the migration has already been applied to the database, first roll back the database to a previous migration before removing it.

4. Updating Existing Migrations
Migrations themselves are generally not edited manually.

The recommended workflow is:

Modify your entity classes.

Create a new migration.

Apply it to the database.

Example:

You add an Email property.

public class Student
{
    public int Id { get; set; }
    public string Name { get; set; }
    public int Age { get; set; }
    public string Email { get; set; }
}
Create a new migration:

dotnet ef migrations add AddEmailColumn
Apply it:

dotnet ef database update
5. Seeding Data during Migrations
What is Data Seeding?
Data Seeding inserts initial data into the database automatically.

Example:

protected override void OnModelCreating(ModelBuilder modelBuilder)
{
    modelBuilder.Entity<Student>().HasData(
        new Student
        {
            Id = 1,
            Name = "Alice",
            Age = 20
        },
        new Student
        {
            Id = 2,
            Name = "Bob",
            Age = 21
        }
    );
}
Create a migration:

dotnet ef migrations add SeedStudentData
Apply it:

dotnet ef database update
The database will automatically contain:

Id	Name	Age
1	Alice	20
2	Bob	21
6. Managing Database Schema Changes
As your application grows, the database structure changes.

Examples:

Add a new column.

Remove a column.

Rename a column.

Create a new table.

Delete a table.

Workflow
Modify Entity
      │
      ▼
Add Migration
      │
      ▼
Update Database
      │
      ▼
Schema Updated
Example
Initial Table
Id	Name
1	Alice
Add a property:

public string Email { get; set; }
Create migration:

dotnet ef migrations add AddEmail
Update database:

dotnet ef database update
Updated Table
Id	Name	Email
1	Alice	NULL
Useful Migration Commands
Command	Purpose
dotnet ef migrations add InitialCreate	Create a new migration
dotnet ef database update	Apply migrations to the database
dotnet ef migrations remove	Remove the latest migration (if not applied)
dotnet ef migrations list	Display all migrations
