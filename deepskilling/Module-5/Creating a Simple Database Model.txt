1. Defining Entities
What is an Entity?
An Entity is a C# class that represents a table in the database.

Class → Database Table

Object → Table Row

Property → Table Column

Example
public class Student
{
    public int Id { get; set; }
    public string Name { get; set; }
    public int Age { get; set; }
}
This creates a table:

Id	Name	Age
1	Alice	20
2	Bob	21
2. Relationships in EF Core
A relationship defines how two tables are connected.

There are three types:

A) One-to-One (1:1)
One person has one passport.

Person  -------- Passport
   1      :      1
B) One-to-Many (1:M) Most Common
One department has many employees.

Department
------------
1   CSE

Employee
------------
1  Ram
2  Ravi
3  Sita

One Department → Many Employees
C) Many-to-Many (M:N)
One student can enroll in many courses.

One course can have many students.

Student <------> Course
EF Core automatically creates a junction table.

3. Primary Key
What is a Primary Key?
A Primary Key (PK) uniquely identifies every row in a table.

Properties named Id or ClassNameId are automatically treated as primary keys.

Example

public class Student
{
    public int Id { get; set; }      // Primary Key
    public string Name { get; set; }
}
Database

Id	Name
1	Alice
2	Bob
Here, Id is the Primary Key.

4. Foreign Key
What is a Foreign Key?
A Foreign Key (FK) links one table to another.

Example

It connects the Student table to the Department table.

5. Navigation Properties
Navigation properties allow moving between related objects without writing SQL joins.

Example

public class Department
{
    public int DepartmentId { get; set; }

    public string Name { get; set; }

    public List<Student> Students { get; set; }
}
public class Student
{
    public int StudentId { get; set; }

    public string Name { get; set; }

    public int DepartmentId { get; set; }

    public Department Department { get; set; }
}
Explanation
Student

DepartmentId
is the Foreign Key.

Department
is the Navigation Property.

Department contains

Students
which is another Navigation Property.

6. Code-First Approach
What is Code-First?
In the Code-First approach, you:

Create C# classes (entities).

Create the DbContext.

Generate migrations.

Create the database automatically.

Workflow
Create Entity Classes
          │
          ▼
Create DbContext
          │
          ▼
Add Migration
          │
          ▼
Update Database
          │
          ▼
Database Created
Example
Step 1: Create Entity
public class Student
{
    public int Id { get; set; }

    public string Name { get; set; }

    public int Age { get; set; }
}
Step 2: Create DbContext
public class AppDbContext : DbContext
{
    public AppDbContext(DbContextOptions<AppDbContext> options)
        : base(options)
    {
    }

    public DbSet<Student> Students { get; set; }
}
Step 3: Create Migration
dotnet ef migrations add InitialCreate
Step 4: Update Database
dotnet ef database update
EF Core creates the database and the Students table automatically.

Advantages of Code-First
No need to manually create tables.

Database is generated from C# classes.

Easy to maintain and modify.

Supports migrations for schema changes.

Best suited for new applications.
