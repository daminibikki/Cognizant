1. Installing EF Core Packages via NuGet
Before using Entity Framework Core (EF Core), install the required NuGet packages.

Using Visual Studio
Right-click the project.

Select Manage NuGet Packages.

Search and install the required packages.

Required Packages

Microsoft.EntityFrameworkCore.Tools

Using Package Manager Console
Install-Package Microsoft.EntityFrameworkCore
Install-Package Microsoft.EntityFrameworkCore.SqlServer
Install-Package Microsoft.EntityFrameworkCore.Tools
Using .NET CLI
dotnet add package Microsoft.EntityFrameworkCore
dotnet add package Microsoft.EntityFrameworkCore.SqlServer
dotnet add package Microsoft.EntityFrameworkCore.Tools
2. Configuring DbContext
What is DbContext?
DbContext is the main class in EF Core that acts as a bridge between your C# application and the database.

It is responsible for:

Connecting to the database

Tracking entity changes

Performing CRUD operations

Managing transactions

Example Entity
public class Student
{
    public int Id { get; set; }
    public string Name { get; set; }
    public int Age { get; set; }
}
Creating DbContext
using Microsoft.EntityFrameworkCore;

public class AppDbContext : DbContext
{
    public AppDbContext(DbContextOptions<AppDbContext> options)
        : base(options)
    {
    }

    public DbSet<Student> Students { get; set; }
}
Explanation
DbContext → Base class for EF Core.

DbSet<Student> → Represents the Students table.

Each DbSet maps to one database table.

3. Connecting to SQL Server
Step 1: Add Connection String
In appsettings.json

{
  "ConnectionStrings": {
    "DefaultConnection": "Server=.;Database=StudentDB;Trusted_Connection=True;TrustServerCertificate=True;"
  }
}
Explanation
Server=.; → Local SQL Server instance.

Database=StudentDB; → Database name.

Trusted_Connection=True; → Windows Authentication.

TrustServerCertificate=True; → Accepts the server certificate.

Step 2: Register DbContext
In Program.cs

using Microsoft.EntityFrameworkCore;

builder.Services.AddDbContext<AppDbContext>(options =>
    options.UseSqlServer(
        builder.Configuration.GetConnectionString("DefaultConnection")));
This tells EF Core to use SQL Server with the specified connection string.

4. Basic EF Core CLI Commands
Add Migration
Creates a migration based on model changes.

dotnet ef migrations add InitialCreate
Example migration names:

InitialCreate

AddStudentTable

AddEmployeeTable

Update Database
Applies all pending migrations to the database.

dotnet ef database update
This creates or updates the database schema.

Remove Last Migration
dotnet ef migrations remove
Removes the latest migration if it has not been applied.

View Migrations
dotnet ef migrations list
Lists all migrations in the project.
