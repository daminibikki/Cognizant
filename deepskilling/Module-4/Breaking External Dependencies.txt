1. Loosely-coupled and Testable Code
What is Coupling?
Coupling means how much one class depends on another.

Tight Coupling (Bad)
class OrderService {
    PaymentService payment = new PaymentService();
}
Problems:

OrderService always uses PaymentService.

Difficult to replace with another payment service.

Hard to unit test.

Loose Coupling (Good)
class OrderService {
    PaymentService payment;

    OrderService(PaymentService payment) {
        this.payment = payment;
    }
}
Now you can pass any implementation.

Advantages:

Easy to test

Easy to maintain

Flexible

2. Refactoring Towards a Loosely Coupled Design
Refactoring means improving the design without changing behavior.

Before
class Student {
    Teacher t = new Teacher();
}
After
class Student {
    Teacher t;

    Student(Teacher t) {
        this.t = t;
    }
}
The behavior is the same, but the code is easier to test.

3. Dependency Injection via Method Parameters
Dependency is passed to a method.

class Printer {

    void print(Document d) {
        System.out.println(d);
    }

}
Every time you call the method, you provide the object.

4. Dependency Injection via Properties (Setter Injection)
Dependency is set using a setter.

class Student {

    Teacher teacher;

    void setTeacher(Teacher teacher) {
        this.teacher = teacher;
    }
}
Usage

Student s = new Student();
s.setTeacher(new Teacher());
5. Dependency Injection via Constructor
Most common approach.

class Student {

    Teacher teacher;

    Student(Teacher teacher) {
        this.teacher = teacher;
    }
}
Usage

Teacher t = new Teacher();
Student s = new Student(t);
Advantages

Dependency cannot be forgotten.

Object is fully initialized.

Preferred in frameworks like Spring.

Comparison
Type	How Dependency is Passed
Method Injection	Method parameter
Property Injection	Setter method
Constructor Injection	Constructor

7. Mocking Frameworks
A mock framework creates fake objects for testing.

Popular frameworks

Mockito (Java)

Moq (.NET)

EasyMock (Java)

Example

Suppose

class PaymentService {
    void pay() {}
}
Instead of using the real payment service, use a fake one in tests.

8. Creating Mock Objects Using Moq
Moq is a mocking framework for C#/.NET, not Java.

Example

var mock = new Mock<IEmailService>();
It creates a fake email service.

In Java, the equivalent framework is Mockito.

Example

EmailService email = Mockito.mock(EmailService.class);
9. State-based vs Interaction Testing
State-based Testing
Checks the final state after a method executes.

Example

list.add(10);
Test

assertEquals(1, list.size());
We only check the result.

Interaction Testing
Checks whether one object called another.

Example

emailService.sendEmail();
Using Mockito

verify(emailService).sendEmail();
We verify the interaction.

Difference
State-based	Interaction
Checks output or object state	Checks method calls
Uses assertions	Uses verify()
Example: List size	Example: Email method called
