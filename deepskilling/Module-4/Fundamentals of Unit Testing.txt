1. Characteristics of Good Unit Tests
A good unit test should be:

✅ Fast
Runs quickly (milliseconds).

Example: Testing a method, not a database.

✅ Independent
One test should not depend on another.

✅ Repeatable
Gives the same result every time.

✅ Self-checking
Automatically tells whether it passed or failed using assertions.

✅ Easy to Read
Test name and code should clearly explain what is being tested.

Example:

@Test
void testAddition() {
    assertEquals(5, add(2,3));
}
2. What to Test
Test:

Normal cases

Boundary values

Invalid inputs

Edge cases

Example

int square(int n)
Test cases:

square(5) = 25
square(0) = 0
square(-3) = 9
3. What Not to Test
Don't test:

Java library methods (String.length())

Third-party libraries

Getter/setter methods (unless they contain logic)

Code that never changes

Example

No need to test:

String name="John";
name.length();
Java already guarantees it works.

4. Naming and Organizing Tests
Use meaningful names.

Good

testLoginWithValidPassword()
Bad

test1()
Group related tests in one class.

Example

CalculatorTest

testAddition()
testSubtraction()
testMultiplication()
5. Black-box Testing
In black-box testing:

We don't care about the code inside.

We only check input and output.

Example

Method

int add(int a,int b)
Test

add(3,5)
Expected

8
We don't care how add() is implemented.

6. Set Up and Tear Down
Sometimes many tests need the same object.

Instead of creating it every time:

Calculator c = new Calculator();
JUnit provides:

@BeforeEach
Runs before every test.

@BeforeEach
void setup() {
    c = new Calculator();
}
@AfterEach
Runs after every test.

Useful for

Closing files

Closing database connections

Cleaning resources

@AfterEach
void cleanup() {
    System.out.println("Finished");
}
7. Parameterized Tests
Same test with multiple inputs.

Instead of

assertEquals(4,square(2));
assertEquals(9,square(3));
assertEquals(16,square(4));
Use

@ParameterizedTest
@ValueSource(ints={2,3,4})
void testSquare(int n){
    assertTrue(square(n)>0);
}
Runs three times automatically.

8. Ignoring Tests
Sometimes a test is incomplete or has a known issue.

Use

@Disabled
@Test
void testDatabase(){
}
JUnit skips this test.

9. Writing Trustworthy Tests
A trustworthy test should:

Test only one thing.

Give consistent results.

Not depend on the internet or current date/time unless mocked.

Have the correct expected value.

Fail when the code is wrong and pass when the code is correct.

Bad example:

Random random = new Random();
assertEquals(5, random.nextInt());
This test may pass or fail randomly.

Good example:

assertEquals(25, square(5));
Always gives the same result.
