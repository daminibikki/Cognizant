1. Testing Strings
Test methods that return a String.

Example
public String greet() {
    return "Hello";
}
Test
@Test
void testGreet() {
    assertEquals("Hello", greet());
}
Other assertions:

assertTrue(name.startsWith("D"));
assertFalse(name.isEmpty());
assertNotNull(name);

Testing Arrays

Test

Method

4. Testing Void Methods
Void methods don't return anything.

Example

public void print() {
    System.out.println("Hello");
}
Usually we verify its effect.

Example

public void add(List<Integer> list){
    list.add(10);
}
Test

@Test
void testAdd() {
    List<Integer> list = new ArrayList<>();

    add(list);

    assertEquals(1, list.size());
    assertTrue(list.contains(10));
}
5. Testing Methods that Throw Exceptions
Example

public int divide(int a,int b){
    return a/b;
}
If

b = 0
Java throws

ArithmeticException
JUnit

@Test
void testException() {
    assertThrows(
        ArithmeticException.class,
        () -> divide(10,0)

}
6. Testing Private Methods
Example

private int square(int n){
    return n*n;
}
Normally do not test private methods directly.

Instead, test the public method that uses them.

Example

public int calculate(int x){
    return square(x);
}
Test

@Test
void testCalculate(){
    assertEquals(25, calculate(5));
}
Why?

Private methods are implementation details.

Users call public methods, not private ones.

7. Code Coverage
Code coverage tells you how much of your code is tested.

Example

if(age>=18){
    return "Adult";
}
else{
    return "Minor";
}
If you only test

age = 20
then only the Adult branch is tested.

Coverage is only 50%.

If you also test

age = 15
then both branches are executed.

Coverage becomes 100% for this method.

Types of Coverage
Line Coverage – How many lines were executed.

Branch Coverage – Whether every if/else path was tested.

Method Coverage – Whether every method was called.

Tools like JaCoCo are commonly used with JUnit to generate coverage reports.
