
Automated Testing
Definition:
Automated testing is the process of using software tools to execute test cases automatically, compare actual results with expected results, and generate reports without manual intervention.

Example:
Instead of manually checking whether a login page works after every code change, an automated test logs in automatically and verifies the result.

2. Benefits of Automated Testing
Saves Time – Tests run much faster than manual testing.

Improves Accuracy – Eliminates human errors.

Reusable – Test scripts can be executed multiple times.

Early Bug Detection – Finds bugs during development.

Supports Continuous Integration (CI/CD) – Tests run automatically whenever code changes.

Better Test Coverage – Thousands of test cases can be executed.

Cost Effective – Reduces long-term testing costs.

3. Types of Tests
a) Unit Testing
Tests a single function or method.

Done by developers.

Example: Testing an Add() method.

b) Integration Testing
Tests interaction between multiple modules.

Example: Login page interacting with the database.

c) System Testing
Tests the complete application.

Ensures the entire system works correctly.

d) Acceptance Testing
Performed by customers or QA.

Verifies that the software meets business requirements.

e) Regression Testing
Ensures existing features still work after code changes.

f) Performance Testing
Measures speed, responsiveness, and stability.

g) Smoke Testing
Basic testing to check whether the application is stable enough for further testing.

4. Test Pyramid
The Test Pyramid is a guideline that suggests having more low-level tests and fewer high-level tests.

        UI Tests
      (Very Few)
     -------------
   Integration Tests
      (Some)
   -----------------
      Unit Tests
      (Maximum)
Unit Tests
Fast

Cheap

Easy to maintain

Integration Tests
Verify interaction between components.

Why use the Test Pyramid?

Faster execution

Easier maintenance

Better reliability

6. Using NUnit in Visual Studio
NUnit is a unit testing framework for .NET applications.

Steps
Open Visual Studio.

Create a C# project.

Add a NUnit Test Project or install the NUnit NuGet package.

Write test methods.

Build the project.

Open Test Explorer.

Run all tests.

View Passed/Failed results.

Example
using NUnit.Framework;

[TestFixture]
public class CalculatorTest
{
    [Test]
    public void AddTest()
    {
        int result = 5 + 3;
        Assert.AreEqual(8, result);
    }
}
Common NUnit Attributes
Attribute	Purpose
[TestFixture]	Marks a test class
[Test]	Marks a test method
[SetUp]	Runs before each test
[TearDown]	Runs after each test
[OneTimeSetUp]	Runs once before all tests
[OneTimeTearDown]	Runs once after all tests
7. Test-Driven Development (TDD)
Definition:
Test-Driven Development (TDD) is a software development approach where developers write tests before writing the actual code.

TDD Cycle (Red → Green → Refactor)
Step 1: Red
Write a test.

The test fails because the feature doesn't exist.

Step 2: Green
Write the minimum code required to make the test pass.

Step 3: Refactor
Improve and clean the code without changing functionality.

Run tests again to ensure everything still passes.

Write Test
     ↓
 Test Fails (Red)
     ↓
Write Code
     ↓
 Test Passes (Green)
     ↓
Refactor Code
     ↓
Repeat
Advantages of TDD
Produces higher-quality code.

Finds bugs early.

Improves code design.

Makes future changes safer.

Encourages better code coverage.

Disadvantages of TDD
Requires extra time initially.

Has a learning curve.

Writing tests for complex features can be challenging.
