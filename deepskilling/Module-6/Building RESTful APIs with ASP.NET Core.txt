1. Creating Controllers
A Controller is a class that handles HTTP requests and returns responses.

It acts as the bridge between the client and the database/business logic.

Example:

using Microsoft.AspNetCore.Mvc;

namespace StudentAPI.Controllers
{
    [ApiController]
    [Route("api/[controller]")]
    public class StudentController : ControllerBase
    {
    }
}
Explanation
ApiController → Marks the class as a Web API controller.

ControllerBase → Base class for API controllers.

Route("api/[controller]") → Defines the URL.

If the controller name is StudentController, then the route becomes:

https://localhost:5001/api/student
2. Actions
An Action is a method inside a controller that handles a specific HTTP request.

Example:

[HttpGet]
public string GetMessage()
{
    return "Welcome to ASP.NET Core Web API";
}

3. Routes
A Route tells ASP.NET Core which action should execute for a given URL.

Example

[Route("api/[controller]")]
If the controller is

StudentController
URL becomes

api/student
Route with Parameter
[HttpGet("{id}")]
public string GetStudent(int id)
{
    return $"Student ID = {id}";
}
Request

GET /api/student/5
Response

Student ID = 5
Common Route Attributes
Attribute	Purpose
[HttpGet]	Read data
[HttpPost]	Insert data
[HttpPut]	Update data
[HttpDelete]	Delete data
4. CRUD Operations using Entity Framework Core
CRUD means

C → Create

R → Read

U → Update

D → Delete

Suppose we have a model.

public class Student
{
    public int Id { get; set; }

    public string Name { get; set; }

    public int Age { get; set; }
}
DbContext
public class AppDbContext : DbContext
{
    public AppDbContext(DbContextOptions<AppDbContext> options)
        : base(options)
    {
    }

    public DbSet<Student> Students { get; set; }
}
DbSet<Student> represents the Students table.

CREATE (POST)
[HttpPost]
public IActionResult AddStudent(Student student)
{
    _context.Students.Add(student);
    _context.SaveChanges();

    return Ok(student);
}
Request

{
  "name":"Rahul",
  "age":22
}
READ (GET All)
[HttpGet]
public IActionResult GetStudents()
{
    return Ok(_context.Students.ToList());
}
Response

[
   {
      "id":1,
      "name":"Rahul",
      "age":22
   }
]
READ by ID
[HttpGet("{id}")]
public IActionResult GetStudent(int id)
{
    var student = _context.Students.Find(id);

    if(student == null)
        return NotFound();

    return Ok(student);
}
UPDATE (PUT)
[HttpPut("{id}")]
public IActionResult UpdateStudent(int id, Student student)
{
    var data = _context.Students.Find(id);

    if(data == null)
        return NotFound();

    data.Name = student.Name;
    data.Age = student.Age;

    _context.SaveChanges();

    return Ok(data);
}
DELETE
[HttpDelete("{id}")]
public IActionResult DeleteStudent(int id)
{
    var data = _context.Students.Find(id);

    if(data == null)
        return NotFound();

    _context.Students.Remove(data);
    _context.SaveChanges();

    return Ok("Deleted Successfully");
}

5. JSON Formatting
Most ASP.NET Core Web APIs send and receive data in JSON (JavaScript Object Notation).

Example Student object

{
   "id":1,
   "name":"Rahul",
   "age":22
}
JSON Rules

Data is stored as key-value pairs.

Keys are enclosed in double quotes (").

Strings use double quotes.

Arrays use square brackets ([]).

Objects use curly braces ({}).

Example Array

[
   {
      "id":1,
      "name":"Rahul"
   },
   {
      "id":2,
      "name":"Priya"
   }
]
6. Serialization and Deserialization
Serialization
Serialization converts a C# object into JSON before sending it to the client.

C# Object

Student s = new Student
{
    Id = 1,
    Name = "Rahul",
    Age = 22
};
Converted to JSON

{
   "id":1,
   "name":"Rahul",
   "age":22
}
Deserialization
Deserialization converts incoming JSON into a C# object.

JSON received

{
   "name":"Rahul",
   "age":22
}
Converted to

Student student
ASP.NET Core performs this automatically when you use a parameter like:

[HttpPost]
public IActionResult AddStudent(Student student)
{
}
