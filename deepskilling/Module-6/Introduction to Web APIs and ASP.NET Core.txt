1. Overview of Web APIs
A Web API (Application Programming Interface) allows different applications to communicate over the internet using HTTP.

For example:

Mobile App ↔ Web API ↔ Database

React Website ↔ Web API ↔ SQL Server

Example:
A food delivery app requests restaurant details.

Client ----HTTP Request----> Web API ----> Database
Client <---HTTP Response---- Web API <----
The API receives the request, processes it, and sends back data (usually JSON).

Why use Web APIs?
Connect frontend and backend

Share data between applications

Build mobile and web applications

Enable third-party integrations

2. REST vs SOAP
Feature	REST	SOAP
Full Form	Representational State Transfer	Simple Object Access Protocol
Data Format	JSON (mostly)	XML only
Speed	Faster	Slower
Simplicity	Easy	Complex
Protocol	Uses HTTP	HTTP, SMTP, TCP, etc.
Flexibility	High	Less
Popularity	Very popular	Mostly enterprise systems
REST Example
Request

GET /api/students/1
Response

{
   "id":1,
   "name":"John"
}
SOAP Example
Request

<soap:Envelope>
   <soap:Body>
      <GetStudent>
         <Id>1</Id>
      </GetStudent>
   </soap:Body>
</soap:Envelope>
SOAP responses are also XML.

Which one is preferred?
Nowadays, most companies use REST APIs because they are:

Faster

Easier to develop

Lightweight

JSON is easy to understand

ASP.NET and Web Development

Step 3: Verify Installation
Open Command Prompt.

4. Create an ASP.NET Core Web API Project
Using Visual Studio

File
 → New
 → Project
 → ASP.NET Core Web API
Choose

Framework: .NET 8

Authentication: None

Enable OpenAPI (Swagger): Yes

Click Create.

Using Command Line

dotnet new webapi -n StudentAPI
Run

cd StudentAPI

dotnet run
Open browser

https://localhost:5001/swagger
Swagger opens and shows all API endpoints.

Controllers
Contains API endpoints.

Example:

[ApiController]
[Route("api/[controller]")]
public class StudentController : ControllerBase
{
}
Models
Represents data objects.

public class Student
{
    public int Id { get; set; }

    public string Name { get; set; }

    public int Age { get; set; }
}
Program.cs
Application entry point.

Registers services and middleware.

Example:

var builder = WebApplication.CreateBuilder(args);

builder.Services.AddControllers();

builder.Services.AddEndpointsApiExplorer();

builder.Services.AddSwaggerGen();

var app = builder.Build();

app.UseSwagger();

app.UseSwaggerUI();

app.MapControllers();

app.Run();
appsettings.json
Stores configuration values.

Example:

{
  "ConnectionStrings": {
    "DefaultConnection":
    "Server=.;Database=StudentDB;Trusted_Connection=True;"
  }
}
