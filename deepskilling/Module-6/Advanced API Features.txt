1. Attribute Routing
Attribute Routing means defining routes directly on controllers or action methods using attributes like [Route], [HttpGet], [HttpPost], etc.

Instead of defining routes in a central configuration file, the route is written close to the action that handles it.

Example
[ApiController]
[Route("api/[controller]")]
public class StudentController : ControllerBase
{
    [HttpGet]
    public IActionResult GetAll()
    {
        return Ok();
    }
}
URL:

GET /api/student
Route Parameters
Route parameters are values passed as part of the URL.

Example:

[HttpGet("{id}")]
public IActionResult GetStudent(int id)
{
    return Ok($"Student Id = {id}");
}
Request

GET /api/student/5
Output

Student Id = 5
Multiple Route Parameters
[HttpGet("{id}/{name}")]
public IActionResult GetStudent(int id, string name)
{
    return Ok($"{id} {name}");
}
Request

GET /api/student/10/Rahul
Output

10 Rahul
2. Query Parameters
A query parameter is passed after the ? in the URL.

Example URL

GET /api/student?age=20
Controller

[HttpGet]
public IActionResult GetStudentByAge(int age)
{
    return Ok(age);
}
Output

20
Multiple Query Parameters
URL

GET /api/student?name=Rahul&age=22
Controller

[HttpGet]
public IActionResult Search(string name, int age)
{
    return Ok($"{name} {age}");
}
Output

Rahul 22
Route Parameter vs Query Parameter
Route Parameter	Query Parameter
/student/10	/student?id=10
Used to identify a resource	Used to filter or search
Required	Often optional
3. Middleware
Middleware is software that executes between the incoming request and the outgoing response.

Every HTTP request passes through a pipeline of middleware components.

Request Flow
Client
   ↓
Authentication
   ↓
Authorization
   ↓
Logging
   ↓
Controller
   ↓
Response

Common Middleware
app.UseHttpsRedirection();

app.UseAuthentication();

app.UseAuthorization();

app.MapControllers();
Explanation
UseHttpsRedirection() → Redirects HTTP requests to HTTPS.

UseAuthentication() → Verifies the user's identity.

UseAuthorization() → Checks whether the user has permission.

MapControllers() → Maps controller endpoints.

4. Custom Middleware
You can create your own middleware for tasks like logging or request timing.

Example

public class LoggingMiddleware
{
    private readonly RequestDelegate _next;

    public LoggingMiddleware(RequestDelegate next)
    {
        _next = next;
    }

    public async Task Invoke(HttpContext context)
    {
        Console.WriteLine("Request Started");

        await _next(context);

        Console.WriteLine("Request Completed");
    }
}
Register it in Program.cs

app.UseMiddleware<LoggingMiddleware>();
5. Filters
Filters allow you to execute code before or after an action method.

Common uses:

Logging

Validation

Exception handling

Authorization

Types of Filters
Filter	Purpose
Authorization Filter	Checks permissions
Action Filter	Runs before/after an action
Exception Filter	Handles exceptions
Result Filter	Runs before/after returning a response
Custom Action Filter
public class MyActionFilter : ActionFilterAttribute
{
    public override void OnActionExecuting(ActionExecutingContext context)
    {
        Console.WriteLine("Before Action");
    }

    public override void OnActionExecuted(ActionExecutedContext context)
    {
        Console.WriteLine("After Action");
    }
}
Use it

[MyActionFilter]
[HttpGet]
public IActionResult GetStudents()
{
    return Ok();
}
6. JWT Authentication
JWT (JSON Web Token) is a secure way to authenticate users.

Instead of storing login information on the server, the server issues a signed token that the client sends with future requests.

JWT Flow
User
   │
Login (username/password)
   │
Server verifies credentials
   │
Generates JWT Token
   │
Client stores token
   │
Client sends token
Authorization: Bearer Token
   │
Server validates token
   │
Access Granted
JWT Structure
A JWT has three parts:

Header
.
Payload
.
Signature
Example

xxxxx.yyyyy.zzzzz
Header
Contains token type and signing algorithm.

Example

{
  "alg":"HS256",
  "typ":"JWT"
}
Payload
Contains user information (claims).

Example

{
   "name":"Rahul",
   "role":"Admin"
}
Signature
Used to verify that the token has not been modified.

7. Configure JWT Authentication
Install the JWT authentication package.

In Program.cs

builder.Services.AddAuthentication(JwtBearerDefaults.AuthenticationScheme)
    .AddJwtBearer(options =>
    {
        // JWT configuration
    });
Enable middleware

app.UseAuthentication();

app.UseAuthorization();
Important: UseAuthentication() must come before UseAuthorization().

8. Protecting an API
Use the [Authorize] attribute.

[Authorize]
[HttpGet]
public IActionResult GetStudents()
{
    return Ok();
}
Only authenticated users can access this endpoint.

Allow Anonymous Access
[AllowAnonymous]
[HttpPost]
public IActionResult Login()
{
    return Ok();
}
Anyone can access the login endpoint.

9. Authentication vs Authorization
Authentication	Authorization
Verifies who the user is	Verifies what the user is allowed to do
Login process	Permission check
Example: Username and password	Example: Admin-only access
10. Bearer Token Example
Request

GET /api/student
Authorization: Bearer eyJhbGciOiJIUzI1NiIs...

200 OK
If the token is invalid or missing:
