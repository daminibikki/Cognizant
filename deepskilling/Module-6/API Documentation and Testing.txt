1. Swagger / OpenAPI
Swagger is a tool used to document, visualize, and test Web APIs.

OpenAPI is the specification (standard), and Swagger is a set of tools that implement that specification.

With Swagger, you can:

View all API endpoints

Test APIs directly from the browser

See request and response formats

Understand API documentation easily

Why Use Swagger?
Automatically generates API documentation

Easy API testing

Shows HTTP methods (GET, POST, PUT, DELETE)

Displays request and response models

Helps frontend and backend developers collaborate

Swagger Flow
Developer
     │
ASP.NET Core Web API
     │
Swagger/OpenAPI
     │
Interactive API Documentation
2. Enable Swagger in ASP.NET Core
In Program.cs

var builder = WebApplication.CreateBuilder(args);

builder.Services.AddControllers();

builder.Services.AddEndpointsApiExplorer();

builder.Services.AddSwaggerGen();

var app = builder.Build();

app.UseSwagger();

app.UseSwaggerUI();

app.MapControllers();

app.Run();
Explanation
AddSwaggerGen() → Generates OpenAPI documentation.

UseSwagger() → Creates the Swagger JSON endpoint.

UseSwaggerUI() → Opens the interactive Swagger webpage.

4. OpenAPI
OpenAPI is a specification that describes REST APIs in a machine-readable format.

It includes:

Endpoints

HTTP methods

Parameters

Request body

Responses

Authentication

Swagger uses this specification to generate interactive documentation.

5. Using Postman
Postman is a popular tool for testing APIs manually.

It allows you to:

Send HTTP requests

View responses

Test authentication

Save requests in collections

Automate simple tests

Postman Interface
Method → URL → Headers → Body → Send
Example

GET

https://localhost:5001/api/student
Click Send.

Response

[
   {
      "id":1,
      "name":"Rahul"
   }
]

9. REST Client
REST Client is a Visual Studio Code extension that lets you send HTTP requests directly from .http or .rest files.

Advantages:

No need to switch to Postman

Store requests with your project

Easy to version-control with Git

Example .http File
GET https://localhost:5001/api/student
Accept: application/json
Click Send Request in VS Code to execute it.

POST Request
POST https://localhost:5001/api/student
Content-Type: application/json

{
   "name":"Rahul",
   "age":22
}

Swagger vs Postman vs REST Client
Feature	Swagger	Postman	REST Client
Built into ASP.NET Core	✅ Yes	❌ No	❌ No
Interactive Documentation	✅ Yes	❌ No	❌ No
Manual Testing	✅ Yes	✅ Yes	✅ Yes
Save Requests	❌ Limited	✅ Yes	✅ Yes (.http files)
Team Collaboration	Limited	Excellent	Good (Git-friendly)
Best Use	API documentation and quick testing	Manual API testing	Testing APIs from VS Code
